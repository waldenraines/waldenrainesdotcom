Subrosa

Subrosa is an online system for hosting real-life games of Assassins.

Players visit the website to sign up and get assigned a kill code and a target. Once the game begins, players hunt each other with Game Master defined "weapons" (in the case of Raleigh Wars we used water "weapons"). A player eliminates another player with one of these weapons and obtains his or her kill code. The successful Assassin records their elimination by entering the victim's kill code (online or via text) and they receive their next target. The game goes on until only one player remains.

Subrosa also provides various assignment types (circular, bi-directional circular, and everyone for themselves), player statistics and accolades, the ability to restrict game entry based on a player's residence, and checkpoints that require a player to be in a location at a certain time. The goal is to provide everything you would need to run a successful game of Assassins.

Technology Used

Keep in mind this web application is about five years old now and in the process of being replaced.

  • Pylons: a Python MVC framework.
  • PostgresSQL for the database layer.
  • RabbitMQ and Celery as a distributed message queue for scheduling and executing game events.
  • Google Maps API for map related functionality.
  • Tropo as SMS API integration; we send and receive SMS messages that affect changes in the game.
  • PayPal API (unfortunately the "classic" version, as this was created before the REST API was available) for payment processing.

Lessons Learned

  • Separate Concerns: Subrosa currently has the API and website bundled together and parts of the application that shouldn't depend on each other do.
  • Test Coverage: Subrosa has very little in the way of unit, integration, or other test coverage. You have to write tests as you write code or they will never get written.
  • Icon Overload: Using images for icons can really slow down page load time when you have a lot of them. Better to use font icons. If you have to use images, use sprites.
  • Use Semantic HTML: Subrosa currently is basically just a mess of divs. Creating a div and styling it to display inline is bad.
  • Style.css is Bad: Having a multi-hundred line CSS file is great when a minifier spits it out but difficult to grok otherwise.
  • No More Spaghetti Javascript: Subrosa has bits of it all over the place. Javscript should be written in reusable modules just like anything else.

The Future

We are currently rewriting the API in Java. We have just begun rewriting the web client as an AngularJS with other device native clients to come.

Zadats

Zadats is a service that provides data sharing between law enforcement agencies. I was part of this project in my spare time for a couple of weeks and completed a Link Analysis feature whereby related people, places, or property are shown in graphical form to help law enforcement discern connections between them in order to solve crimes.

Technology Used

  • AngularJS for keeping the javascript code sane and tested.
  • D3.js for creating a force-directed graph.
